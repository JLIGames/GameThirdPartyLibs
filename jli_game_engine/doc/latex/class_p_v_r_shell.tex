\subsection{Detailed Description}
Inherited by the application; responsible for abstracting the O\+S and A\+P\+I. 





\hyperlink{class_p_v_r_shell}{P\+V\+R\+Shell} is the main Shell class that an application uses. An application should supply a class which inherits \hyperlink{class_p_v_r_shell}{P\+V\+R\+Shell} and supplies implementations of the virtual functions of \hyperlink{class_p_v_r_shell}{P\+V\+R\+Shell} (\hyperlink{class_p_v_r_shell_a1ecb1d7021a3cc3f533acddc5ba8c1ce}{Init\+Application()}, \hyperlink{class_p_v_r_shell_adeebed7c8025a7270cee6f9609a97d68}{Quit\+Application()}, \hyperlink{class_p_v_r_shell_ad1b978ddb9fb35b1d426818b756230fd}{Init\+View()}, \hyperlink{class_p_v_r_shell_a65a1e2efc768686903fd30c56218ed02}{Release\+View()}, \hyperlink{class_p_v_r_shell_ae0eb5f797cbe993a22b8659f9c332578}{Render\+Scene()}). Default stub functions are supplied; this means that an application is not required to supply a particular function if it does not need to do anything in it. The other, non-\/virtual, functions of \hyperlink{class_p_v_r_shell}{P\+V\+R\+Shell} are utility functions that the application may call.

It is called any time the rendering A\+P\+I is initialised, i.\+e. once at the beginning, and possibly again if the resolution changes, or a power management even occurs, or if the app requests a re-\/initialisation. The application should check here the configuration of the rendering A\+P\+I; it is possible that requests made in \hyperlink{class_p_v_r_shell_a1ecb1d7021a3cc3f533acddc5ba8c1ce}{Init\+Application()} were not successful.
