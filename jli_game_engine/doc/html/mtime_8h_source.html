74  // Time Class
75  // R. Hoetzlein
76  //
77  // Overview:
78  // There is a need in many systems to represent both very small (nanoseconds) and
79  // very large (millenia) timescales accurately. Modified Julian Date accurate represents
80  // individual days over +/- about 30,000 yrs. However, MJD represents fractions of a day
81  // as a floating point fraction. This is inaccurate for any timing-critical applications.
82  // The Time class here uses an 8-byte (64 bit) integer called SJT, Scaled Julian Time.
83  // SJT = MJD * DAY_SCALAR + UT (nanoseconds).
84  // SJT is the Modified Julian Date scaled by a integer factor, and added to Universal Time
85  // represented in nanoseconds.
86  //
87  // Features:
88  // - Accurately represents individual nanoseconds over +/- 30,000 yrs.
89  // - Correct rollover of tiny time scales on month, day, year boundaries.
90  // e.g. Set date/time to 11:59:59.9999, on Feb 28th,
91  // - Accurately gives day of the week for any date
92  // - Accurately compares two dates (days elapsed) even across leap-years.
93  // - Adjust sec/nsec independently from month/day/year (work at scale you desire)
94  //
95  // Implementation Notes:
96  // JD = Julian Day is the number of days elapsed since Jan 1, 4713 BC in the proleptic Julian calendar.
97  // http://en.wikipedia.org/wiki/Julian_day
98  // MJD = Modified Julian Date. Most modern dates, after 19th c., have Julian Date which are greater
99  // than 2400000.5. MJD is an offset. MJD = JD - 2400000.5
100  // It shifts the epoch date (start date) to Nov 17, 1858.
101  // UT = Universal Time. This is the time of day in hours as measured from Greenwich England.
102  // For non-astronomic uses, this is: UT = Local Time + Time Zone.
103  // SJT = Scaled Julian Time = MJD * DAY_SCALAR + UT (in nanoseconds).
104  //
105  // Julian Dates (and their MJD and SJT equivalents)
106  // ------------
107  // Jan 1, 4713 BC = JD 0 = MJD -2400000 = SJT
108  // Jan 1, 1500 AD = JD 2268933.5 = MJD -131067 = SJT
109  // Nov 16, 1858 AD = JD 2400000.5 = MJD 0 = SJT 0
110  // Jan 1, 1960 AD = JD 2436935.5 = MJD 36935 = SJT 3,191,184,000,000
111  // Jan 1, 2005 AD = JD 2453372.5 = MJD 53372 = SJT 4,611,340,800,000
112  // Jan 1, 2100 AD = JD 2488070.5 = MJD 88070 = SJT 7,609,248,000,000
113  //
114  //
115  //
116  // 32/64-Bit Integer Ranges
117  // 32-bit Integer Min: –2,147,483,648 ( 4 bytes )
118  // 32-bit Integer Max: 2,147,483,647
119  // SJT 2005: 4,611,340,800,000
120  // 64-bit Integer Min: –9,223,372,036,854,775,808
121  // 64-bit Integer Max: 9,223,372,036,854,775,807 ( 8 bytes )
122  //
123  // SJT Range
124  // ---------
125  // * USING DAY_SCALAR = 86,400,000 (millisec accuracy)
126  // SJT Range = (+/-9,223,372,036,854,775,807 SJT / 86,400,000 DAY_SCALAR)
127  // SJT Range (in Julian Days) = +2400000.5 + (+/-106,751,991,167 MJD)
128  // SJT Range (in Julian Days) = +/- 292278883 years, with 1 millisecond accuracy.
129  //
130  // * USING DAY_SCALAR = 86,400,000,000,000 (nanosec accuracy)
131  // SJT Range = (+/-9,223,372,036,854,775,807 SJT / 86,400,000,000,000 DAY_SCALAR)
132  // SJT Range (in Julian Days) = +2400000.5 + (+/-106,751 MJD)
133  // SJT Range (in Julian Days) = 1566 AD to 2151 AD, with 1 nanosecond accuracy.
