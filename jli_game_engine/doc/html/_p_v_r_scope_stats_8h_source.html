181 /*!**************************************************************************
182  @brief Call regularly to allow PVRScope to track the latest hardware
183  performance data. If psReading is not NULL, PVRScope will also
184  calculate and return counter values to the application.
185  @details This function should be called "regularly"; two use cases are
186  considered:
187  1) A 3D application rendering a performance HUD (e.g. the on-
188  screen graphs in PVRScopeExample). Such an application should
189  call this function at least once per frame in order to gather
190  new counter values. If slower HUD updates are desired,
191  psReading may be NULL until a new reading is required, in
192  order to smooth out values across longer time periods, perhaps
193  a number of frames.
194  2) A standalone performance monitor (e.g. PVRMonitor) or
195  logging application. Such an application should idle and
196  regularly wake up to call this function; suggested rates are
197  100Hz (10ms delays) or 200Hz (5ms delays). If counter updates
198  are required at a lower rate, set psReading to NULL on all
199  calls except when new counter values are desired.
200 ****************************************************************************/
201 int PVRScopeReadCounters(
202  struct SPVRScopeImplData * const psData,
